网易京东360蘑菇街春招感触

发表于 2017-05-06   |  

前言

自己之前就决定了毕业找工作的道路，所以一直关注着互联网公司春秋季校园招聘。作为18届毕业生，也差不多在这个时候准备找实习了。从开始准备笔试，面试，到现在拿到offer，整个春招，给自己带了来了很多感受，无论是技术上还是为人上。既然我的春招告一段落，每次面试也都尝试着去总结面试官的问题，因此，就写下了这些文字，勉励自己，同时也给秋招做准备。

基本情况

春招开始到现在，我将投过的公司和最后的应聘结果整理了一下，大概投了10来家公司，拿到两个offer。关于我应聘的岗位，因为之前的项目经验在python后台开发，然而根据春招的情况，大型互联网公司并不会单独招python岗，于是抱着“曲线救国”的想法，先找到实习再说，我就投了一些测试开发的岗位，因为如果做自动化测试，性能测试，一定程度也能提升自己的编码能力，而且python很适合编写这类的测试脚本。当然，现在自己也有在考虑往java后台开发方向转准备秋招，下面是投递的情况：

  • 美团点评内推 （笔试未通过）
  • 360网申 4月视频面试 3面 （成为备胎，最后也挂了）
  • 网易内推 4月杭州现场面试 3面 （offer）
  • 京东网申 4月电话面试 3面 （offer）
  • 蘑菇街内推 4月底开始视频面试 （就在写这篇文章的时候，蘑菇街hr打电话过来说二面过了，预约了hr面）
  • 七牛云存储网申 （简历未通过）
  • 招商银行信用卡中心 （简历未通过)
  • 去哪儿网申 （笔试未通过）
  • 美图今日头条 （时间冲突未参加笔试），此外还投了诸如百度散招，都没有结果

春招准备

上学期末考试的时候开始收集春招的资料，针对自己理想的岗位准备复习计划。要复习的知识无非就是计算机网络，数据结构算法，操作系统，数据库之类的基础知识，还有必须的一项是要熟悉一门开发语言，再者就是针对自己岗位做知识储备了。前面的大学基础知识无论是应聘什么岗位都会被面试官问道，所以要好好准备，这些知识除了看书积累没有其它方法，还有需要重点准备的是数据结构与算法，这方面很能体现能力，并且有的公司笔试就考编程题，最后就是自己针对应聘的岗位准备对应的知识就好了。申请互联网公司技术岗，第一关就是笔试，笔试涉及各种基础知识和编程题，笔试没有通过也就没有面试机会了。在这里推荐两个东西，第一个是《剑指offer》，一本堪称神器的书，第二个是牛客网和leetcode平台，牛客网里面有大量的招聘信息，面经和题库，好不好，谁用谁知道！leetcode可以在用来刷编程题，很多面试题都会出自那里。其实我寒假在家的时候就想准备好好看书好好刷题好好睡觉好好写代码(但是实际打了一个寒假的王者荣耀，这游戏有毒，自己也没毅力)，寒假没好好准备，也弄的我春招开始阶段很狼狈。

总结

因为自己来自偏远地区双非学校，春招大军竞争非常激烈，很多211 985的本硕，无形之中就给自己增加了很大的压力。尤其是刚开始的时候，笔试连环挂，没拿到一个面试机会，自己的内心会很奔溃，会怀疑自己。我个人而言，第一场面试360的时候也紧张，早早地在电脑面前等待着面试官上线。当然，应聘技术岗，面试官最看重的还是自己的能力，看你的能力能不能完成公司的既定业务，有没有潜力。来自己不同的学校，本科或者硕士，公司都不会区别对待，这点我感受很深。所以啊，还是要努力提升自己的专业能力，技术方向也没什么捷径可走，多多看书，多多项目实践，多多总结。

Python函数式编程

发表于 2017-03-02   |   分类于 python   |  

1、Haskell到python函数式编程

    在学习python的过程中，一直体会着它的简洁和优雅，也钟情于它的简洁和优雅。但在一次偶然的机会中，在微博上看到某位博主贴出的一串Haskell代码，代码的透明，凝练，模块化，由小到大把我深深的吸引住，了解之余，发现Haskell是一门函数式编程语言，函数式编程语言有着深厚的数学知识作为基础，将电脑的运算看作是数学上的函数计算，可能这也是“数学之美”吧，在震惊之余，我也尝试了解，python是不是也支持函数式编程呢？结果是肯定的，Python作为一门命令式语言，但是也具备了函数式编程的特点。
    经过查询资料，不断学习，也慢慢的发现，函数式编程的确是很深奥，甚至于是一种晦涩难懂的编程模式，它将计算机中的运算看作是数学中函数的运算。掌握了可以让我们受益匪浅,也能体会到python一行代码实现强大功能的乐趣。

2、函数式编程里的数学知识

    函数式编程里面含有很多的数学知识，函数作为函数式编程里面的一等公民，它来源于数学里面函数，而数学中，递归，合成与柯里化，高级函数。这些函数的转化关系，也都体现在了函数式编程中。

3、函数式编程的特点

  • 函数就是数据，函数是一等公民
  • 纯函数:给一个值，一定返回另一个值，运算过程中部会被外部改变，具有透明性
  • 无状态，区别于命令式编程
  • 强编译器的支持（python不支持)

发表于 2016-11-25   |  

1、背景

    之前在完成一个web项目的时候，采用了前后端分离的模式，后端采用了Flask搭建服务，前端基于React框架搭建界面。后台服务部署在了阿里云服务器，使用git来进行项目代码的更新，但是在前端通过Ajax调试后台接口的时候，却一直报错，对于第一次尝试前后分离的我们来说打击不小，最后对错误代码的分析，原来是跨域的问题。

2、什么是跨域

    要说跨域，首先要了解Ajax(Asynchronous JavaScript+XML),它的核心是XMLHttpRequest（XHR）对象，这个技术使得我们的界面无需刷新便可以异步地从服务器获得数据，从而更新DOM。

    跨域给开发者带来了很多的方便，同时也出现了一些弊端，XHR对象是实现Ajax通信的主要方式，而这样就带来了跨域安全的问题。在默认的情况下，XHR对象只能访问与包含它的页面处于同一个域中的资源，也就是在协议，域名，端口完全相同的情况中，是不会出现跨域的问题。这样做的目的是为了保证页面的安全性，防止界面受到外部资源的干扰，比如恶意代码的注入。但是不可避免的，在实际使用中，有时还是需要使用一些其它的外部资源，为了解决这个问题。w3c就制定了一套标准，使得浏览器和服务器之间在跨域可以相互沟通。

3、跨域协商机制

    每一次Ajax请求都会发送HTTP请求，因为，协商机制也就体现在了HTTP报文中，HTTP头部定义了很多可以用来浏览器和服务器协商的字段，通过这些字段，能让服务器对浏览器的请求作出成功或者失败的响应。下面来了解一些浏览器和服务器端经常使用的一些http首部字段：

    浏览器端http报文中一些常见的头部如下：

  • Accept 浏览器能够处理的内容类型，比如Json
  • Accept-Charset 浏览器能够处理的字符类型
  • Accept-Encoding: 能够处理的压缩编码类型，如gzip
  • Accept-Language: 浏览器当前设置的语言
  • Connection: 连接类型，比如keep-alive 长连接
  • Content-type: 内容的类型
  • Host： 目标主机名
  • Origin: 源
  • Referer: 发出请求页面的url

    举个浏览器请求头部的例子

    服务器端http报文中一些常见的响应头部如下：

  • Access-Control-Allow-Origin: 允许哪个域对资源进行访问，允许全部用*
  • Connection连接的类型
  • Content-Encoding: 编码格式，压缩数据有利于减少阐述时间，浏览器接受后解码
  • Server: 服务器类型
  • Vary:主要用户缓存服务器对于内容压缩的使用，服务器使用浏览器能接受的压缩方式

1、flask的阻塞

flask是一个阻塞式的框架，在flask自带的服务器中，一次只能处理一个请求。所以、使用flask开发的网站后台处理网络请求也是这样的，类似于工厂的流水线生产、一个工人只有完成自己手上当前的工作，才能接着完成流水线上过来的另一个工作。

后台开发中，不可避免的会存在一些非常耗时的请求，比如一些IO操作，或者耗时的计算任务。这样导致上一个请求处理完成之前，服务器无法处理其它请求，这样就造成了阻塞。虽然增加线程的数量能在一定程度上缓解这种糟糕的情况，但当并发量非常大时，多线程就显得无能为力。所以，要增加网络的响应速度，处理好耗时的操作是关键。

    在内存很小的情况下，为了避免Linux服务器宕机，设置更大的交换分区可以稍微解决燃眉之急，在设置2G的交换空间之后，情况得到了缓解

2、了解redis

    我们已经知道诸如Mysql，sqlserver，Mongodb这类的数据库，这些数据库都有一个共同点，都是将数据存储于硬盘。数据存储在硬盘中的优点是硬盘空间大，但是读写性能的低下也困扰着开发者，特别是对于一些请求量非常大但是变更很少的数据，读取的成本比较高，如果能进行缓存，可以提升很大的IO效率。

    有现实的需求，自然就有相应的工具实现。redis便是一个基于内存也可以持久化的数据库，将数据存储于内存中(当数据大小没有超过”swappability=age*log(size_in_memory)这个值时，超过了便写入硬盘），大大提高热点数据的读取效率。同时，它不仅支持key/value类型的数据，同时还提供了list，hash，set等数据结构的存储。redis的相关使用以后将会用单独的文章来写，这里就不详细描述了

3、Celery

    Celery是使用python编写的一个分布式系统，可以被很多语言编写的程序调用，进行异步任务队列的分发。异步，简而言之就是程序将当前要做的事情先放在后台，稍后再进行处理，缓解当前的压力。

    Celery中有一个重要的概念叫broker,我们知道，Celery中的任务是异步地移交给后台处理，那么怎么能确保系统能记住这个任务不会忘记呢，答案就是这个broker，broker便是存储任务的地方，一般使用RabbitMQ或者Redis。同时，还有一个概念是backend,Celery的异步任务是不能立即返回结果给请求的，如果需要返回结果，便可以先将结果暂存，下次请求就可以直接返回了。

2015年总结和新年鸡血

发表于 2016-02-18   |   分类于 其它   |  

写这篇文章之前，我自己曾经犹豫过，到底还要不要写。一来16年已经快过去3个月了，二来不知道该写什么，该怎么总结，但是仔细想了想，还是写吧，趁自己对15年发生的事记忆犹新，趁现在还没开学，有时间~当然，博客的目的就是为了记录自己，而最重要的是想在新年给自己打打鸡血

主要在生活，学习，专业上来回顾自己的这一年发生的事。15年对于我来说，充实而又有意义，很多以前没有做过的事情，在15年也尝试的做了，在这一年，学到了更多的知识，认识了更多有趣的人，去了更远的地方。

先来说说生活

15年完成了大一下学期到大二上学期，学弟变成了学长的转变，总体而言，在对大学新鲜感神秘感退却以后，生活也慢慢的回归了平静，把更多的时间放在自己想做的事情上。因为课程的增加，自然没有大一上学期那般悠闲，自己自由的时间在变少，这一点在刚开学的时候就感受到了。虽然时间变少，可自己做的事情却比以前更多了。而且以前自己种的果也要自己吃，刚来大学的时候，自己怀着热情报名参加了很多社团，有一些要面试的社团也神奇的通过了。因此最后，我同时参加了三个社团，校青协、校职协还有计算机科学协会（acs）。对于前两个社团，自己一直都是酱油般的存在，很少参加活动，但是到了后来，自己仔细一想，发现良心上过不去。毕竟当时是自己报名参加的，既然参加了，就应该把该有的职责给承担起来，不能逃避。于是又主动的参加了一些活动，有些活动有意义，但更多的活动是没有意义的、比如让你写个无意义的策划，做个word。正因为社团无聊的活动太多，于是到了后来，大家对社团的热情都不高了，很多活动都没有人参加，这时候我们就被逼着参加，对于逼大家参加活动这种方式，我个人是非常反感的。不过这也让我更加相信了一句话

大学的很多社团都是松散的，无纪律的，虽然社团能成就一些人，可对于更多的人来说，参加社团活动就是浪费时间，完全学不到什么。

但是其中acs却让我感受到了不同，里面的学长学姐不会教你这个社会的”大道理”，而是真诚的对待你，会把生活学习中的曾经遇到过的些坑告诉你。正因为如此，我课余的时间都待在协会这边的工作室，在这里，大家因为志趣相投而聚在一起。也许是自己在这边做事比较认真，也许是其它原因，学长们慢慢地让我去组织一些大型活动。比如6月的校程序设计大赛，自己作为组织者之一，看着活动办好了，意义也很大，心里说不出的高兴。再后来，自己也成为了这个协会的管理者，责任越来越大。但自己管理半年之后，现在再看，凭心而论，自己确实没有把协会带好，这里面有很多的原因，更多的原因是自己，也在趁过年的时间好好反思。
15年自己也去了一些地方，暑假参加学校的三下乡活动，跨院联合20多人到婺源太白镇教小孩子们学习一些知识，短暂的十几天，我们给小镇带去一些活力。活动还是很有意义的，每天去学校的时候，都可以看见小孩子们很开心，帮村民修电脑也是有意义的事情。三下乡活动结束后，自己回家安静了几天，就和朋友们一起去了广州，在那里度过了5天。前几天还在婺源农村，瞬间就到了发达的广州，由破败到金碧辉煌灯红酒绿，自己的心理有很大的落差，有的时候也会感叹为何社会发展差距如此之大（好像忧国忧民也没有用，还是好好学习吧～）。在玩的几天中，更加认识到，想要到大城市立足，自己需要付出很多的努力。就像老话说的那样，付出不一定有收获，但是不付出一定没有收获。
社团生活说了很多，主要是它的确占了我生活很大一部分吧，那啥就不说了，感觉自己是注孤生的存在。。。

再来说说学习

学习上，这一年自己没有认真去看书，上课也不知道自己干了什么，考试也是期末临时突击，这样没有挂科已是万幸。自己清楚以后读研的可能性不大，因此学习上没多花时间我倒也是心安理得。不过，谁能告诉我，课堂的时间我都干什么去了！！！是该好好想一下这个问题，好好的利用课堂时间做点什么，不要白白浪费了自己还不知道。同时，专业主干课该学的还是要认真学，比如计算机网络（虽然这门已经学完了）这些课的很多知识在自己编程开发过程中都会遇到，不能小看。专业学习因为自己刚来的时候嚷嚷着想学硬件，也学习一些知识，用树莓派做了一些东西。但技术含量不高，同时，因为学校缺少这方面的老师，自学资料少，感觉走下去难度太大。因此有意提高自己对Python的运用能力，因为Python能做好多事。经过几个学期的学习，意识到，cs本科的基础课程数据结构/算法对自己来说非常有用，自己也要尝试刷点题目，把基础学好，再针对某一个就业方面深入学习， 才是正道。

最后说说专业

15年上学期自己转了专业，转到了网络工程，原来的专业在学校是个坑啊。现在很庆幸当初有学长提醒自己转专业。原来用树莓派做了一辆小车，匆忙之间拿去参加比赛了，后来下半年也陆陆续续参加了一些专业比赛。虽说这些比赛是省级的，但是都很水。现在很多比赛就看你演讲的怎么样，答辩的效果怎么样，却忽略背后深层次的技术，然后问你的作品“商业价值”“能不能挣钱”这些于技术本身无关的问题，这些比赛都是计算机作品的比赛啊，问这样的问题真的好吗！！现在社会上流行创新创业，这股风气也带进了学校，个人十分反感。虽然大学生头脑灵活点子多，但是被“逼”着创新创业也不太好吧。在比赛过程中，甚至看见过没有作品，只有一个ppt，答辩的时候就说自己的创意的参赛队伍！这还是计算机“作品”比赛吗？当然，也参加过机器人比赛，这种比赛能不能取得好成绩很大程度上取决于你学校肯不肯多投钱，买新的传感器和配件，不是完全看技术的比赛。现在，自己在慢慢的学习web开发。

新年给自己的鸡血

新年，也想大致的给自己定下一些目标，努力完成吧~~

  • 刷题
  • 上半年完成一个项目，目前已有初步计划
  • pyhton web上多花时间
  • 专业课程认真学
  • 参加有意义的比赛（免费旅游涨姿势，拿了奖学金又可以买机械键盘固态硬盘····hahhhhhh
  • 锻炼身体学英语
  • 出去玩玩（一个人的旅行？
